## 手順

1. `git diff HEAD` で変更差分を確認する
2. 変更されたファイルを必要に応じて Read で読み込み、文脈を把握する
3. 以下の観点でレビューする
4. 結果を構造化して報告する

## レビュー観点

**バグ・正確性**
- 論理的な誤り、境界値・エッジケースの漏れ
- 非同期処理の誤用、競合状態

**セキュリティ**
- XSS、SQLインジェクション、CSRF などの脆弱性
- 機密情報のハードコード、不適切な権限設定

**パフォーマンス**
- 不必要な再レンダリング、無駄なループ・クエリ
- 大きなバンドルサイズになる import

**保守性・可読性**
- 複雑すぎる関数・コンポーネント
- 重複コード、命名の不明確さ

## 出力形式

変更ファイルごとにセクションを分けて報告する。

```
### ファイル名

**[重要度: 高/中/低]** 指摘内容

現状:
\`\`\`
該当コード
\`\`\`

修正案:
\`\`\`
修正後コード
\`\`\`
```

最後に **総合評価**（問題なし / 軽微な指摘あり / 要修正）を示す。
重大な問題がある場合はユーザーの承認を得てから修正を適用する。
