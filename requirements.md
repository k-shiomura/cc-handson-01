# ポートフォリオサイト 要件定義書

## 1. プロジェクト概要

個人ポートフォリオサイトの設計・開発に関する要件定義書。

- **種別**: 静的サイト（SPA: Single Page Application）
- **目的**: 自己紹介・スキル・制作物を発信するポートフォリオサイト
- **デプロイ先**: GitHub Pages
- **作成日**: 2026-03-06

---

## 2. 技術スタック

| 分類 | 技術 | バージョン |
|------|------|-----------|
| フレームワーク | React | 18.x |
| ビルドツール | Vite | 5.x |
| CSSフレームワーク | TailwindCSS | 3.x |

## 3. ページ構成・セクション要件

### 3.1 Hero セクション

**目的**: ファーストビューで訪問者の注意を引き、名前・肩書きを伝える

**表示要素**
- 氏名（大見出し）
- 肩書き / キャッチコピー（サブタイトル）
- 自己紹介の短文（1〜2文）
- CTA ボタン（例: 「プロジェクトを見る」「連絡する」）

**動作要件**
- ページロード時にフェードイン / スライドインアニメーションを適用
- CTA ボタンクリックで対象セクションへスムーズスクロール

---

### 3.2 About セクション

**目的**: 自己紹介・経歴・人物像を伝える

**表示要素**
- プロフィール画像
- 自己紹介文（複数段落）
- 基本情報（居住地・職種など）
- SNS / GitHub リンク

---

### 3.3 Skills セクション

**目的**: 保有スキルと習熟度を視覚的に伝える

**表示要素**
- カテゴリ別スキル一覧（例: Frontend / Backend / DevOps / Tools）
- 各スキルのアイコン + ラベル
- 習熟度インジケータ（プログレスバー or バッジ）

**動作要件**
- スクロールで画面内に入ったタイミングでアニメーション開始

---

### 3.4 Projects セクション

**目的**: 制作物・実績を紹介する

**表示要素**
- プロジェクトカード（グリッドレイアウト）
  - サムネイル画像
  - プロジェクト名
  - 概要説明（2〜3文）
  - 使用技術タグ
  - GitHub リンク / デモリンク

**動作要件**
- カードホバー時にシャドウ + 微小な拡大エフェクト
- タグによるフィルタリング機能（任意）

---

### 3.5 Contact セクション

**目的**: 訪問者からの問い合わせ手段を提供する

**表示要素**
- お問い合わせフォーム
  - 名前（必須）
  - メールアドレス（必須）
  - メッセージ（必須）
  - 送信ボタン
- SNS / メールアドレスの直接リンク

**動作要件**
- フォームバリデーション（クライアントサイド）
- 送信先: Formspree / EmailJS 等の外部サービスを使用（バックエンド不要）
- 送信完了後に成功メッセージを表示

---

## 4. 共通 UI 要件

### 4.1 ナビゲーション

- 固定ヘッダー（スクロール時も常時表示）
- セクションリンク（クリックでスムーズスクロール）
- スクロール位置に応じたアクティブリンクのハイライト
- モバイル時はハンバーガーメニューに切り替え

### 4.2 フッター

- コピーライト表示
- SNS リンク（GitHub / Twitter / LinkedIn など）
- ページトップへ戻るボタン

---

## 5. レスポンシブデザイン要件

モバイルファーストで設計し、以下のブレークポイントに対応する。

| ブレークポイント | 幅 | 対象デバイス |
|-----------------|-----|-------------|
| `sm` | 640px〜 | 大型スマートフォン |
| `md` | 768px〜 | タブレット |
| `lg` | 1024px〜 | ノート PC |
| `xl` | 1280px〜 | デスクトップ |

**主な変化点**
- モバイル: 1カラム縦並びレイアウト
- タブレット以上: 2カラム / グリッドレイアウトに切り替え
- Projects カード: モバイル 1列 / タブレット 2列 / PC 3列

---

## 6. ダークモード要件

- OS のカラースキーム設定（`prefers-color-scheme`）をデフォルトとして検出
- ヘッダーにダークモード切り替えトグルボタンを配置
- 選択状態を `localStorage` に保存し、次回アクセス時に維持
- TailwindCSS の `darkMode: 'class'` 戦略を使用

**カラートークン例**

| トークン | ライト | ダーク |
|---------|--------|--------|
| 背景色 | `#ffffff` | `#0f172a` |
| テキスト（主） | `#1e293b` | `#f1f5f9` |
| テキスト（副） | `#64748b` | `#94a3b8` |
| アクセント | `#3b82f6` | `#60a5fa` |
| カード背景 | `#f8fafc` | `#1e293b` |

## 7. パフォーマンス要件

| 指標 | 目標値 |
|------|--------|
| Lighthouse Performance | 90点以上 |
| Lighthouse Accessibility | 90点以上 |
| Lighthouse SEO | 90点以上 |
| First Contentful Paint (FCP) | 1.5秒以内 |
| Largest Contentful Paint (LCP) | 2.5秒以内 |

**最適化方針**
- 画像は WebP 形式を使用し、`loading="lazy"` を適用
- コードスプリッティングは Vite のデフォルト設定を活用
- 不要な依存ライブラリを追加しない

---

## 8. アクセシビリティ要件

- セマンティックな HTML タグを使用（`<header>`, `<main>`, `<section>`, `<footer>` など）
- 画像には `alt` 属性を設定
- キーボード操作でナビゲーション可能
- カラーコントラスト比は WCAG AA 基準（4.5:1 以上）を満たす
- フォーム要素には `label` を関連付ける

---

## 9. SEO 要件

- `<title>` タグと `<meta name="description">` を設定
- OGP タグ（Open Graph Protocol）を設定
  - `og:title`, `og:description`, `og:image`, `og:url`
- `robots.txt` / `sitemap.xml` を配置

---

## 10. デプロイ要件

### 10.1 GitHub Pages 設定

- リポジトリ名: `<username>.github.io` または `portfolio`
- デプロイブランチ: `gh-pages`
- ビルド成果物: `dist/` ディレクトリ

### 10.2 CI/CD

- GitHub Actions を使用してデプロイを自動化
- トリガー: `main` ブランチへの push
- ワークフロー概要:
  1. Node.js セットアップ
  2. 依存関係インストール（`npm ci`）
  3. ビルド（`npm run build`）
  4. `gh-pages` ブランチへデプロイ

### 10.3 Vite 設定

- `vite.config.ts` の `base` オプションをリポジトリ名に合わせて設定
  ```ts
  // 例: リポジトリ名が "portfolio" の場合
  base: '/portfolio/'
  ```

## 12. 開発フロー

1. リポジトリ作成・初期セットアップ（Vite + React + TypeScript テンプレート）
2. TailwindCSS 導入・カラートークン設定
3. ダークモード機能実装
4. 共通コンポーネント（Header / Footer）実装
5. 各セクション実装（Hero → About → Skills → Projects → Contact の順）
6. レスポンシブ対応・アニメーション調整
7. SEO / OGP 設定
8. Lighthouse 計測・パフォーマンス改善
9. GitHub Actions 設定・GitHub Pages デプロイ確認
